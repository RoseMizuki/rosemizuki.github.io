User Research

User Interviews

User interviews were conducted to find out what users would expect when seeing a portfolio site as well as why they would create or use a portfolio website.

Competitor Analysis

A competitor analysis is looking at and analysing another website that has a similar purpose to your own. This is to see what others are doing that is effective or not effective so that improvements can be made. In this case, I analyzed fellow student websites.

Feature Value Matrix

A feature value matrix is matrix that lists the features included in a website, like resume or email address, and rates how important these are to the user, to the business, and how much development effort is needed to implement the feature. The features I have in this matrix are those that are on my website already and those that I feel would be nice to add on after the basic website is running.

Feature Prioritization

I prioritized my features by what would be the basic things needed in a working website. Anything after what is basically needed such as an about page with a short bio and a form of contact, it can be added later on as it isn't needed in the minimal viable product.

User Testing

User Stories

User stories are important as they tell you what a specific type of user would do to look for on the website and why they would look on the website in the first place. It also tells you what kind of event should happen for the user to deem that they were successful in their search.

Cognitive Walkthrough

A cognitive walkthrough is having users perform certain tasks on the website while asking them questions along the way. This is to find any design flaws that could possibly hinder a user's experience when using the website. I particularly had a simple website so the cognitive walkthroughs didn't give me too much information.

User Test

The two tests I used were a questions test and a 5 second test. These were chosen as they are more qualitative tests and can provide insight on how my website is laid out and if it is effective for users. I found that in the 5 second test a lot of users thought that this website was a travel website instead of a professional website due to the content of the screenshot they saw. In the question test, I found that how I laid out my About page was not easy for users to read the content on the page due to spacing.
